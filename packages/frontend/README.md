# フロントエンド README

このリポジトリは、Next.js (React) を使用して構築されたフロントエンドアプリケーションです。バックエンドAPIと連携し、ユーザーにリッチなUIとインタラクティブな体験を提供します。

## ディレクトリ構造と責務

このプロジェクトは、責務に基づいて明確に分離されたディレクトリ構造を持っています。

- `src/app/`
  - **責務**: アプリケーションのルーティングとエントリーポイントを管理します。Next.jsのApp Routerに基づいています。
  - `layout.tsx`: 全てのページで共有されるグローバルなレイアウトを定義します。ヘッダーやフッター、サイドバーなどの共通UIの配置場所です。
  - `page.tsx`: アプリケーションのメインページ（ホームページ）のコンテンツを定義します。
  - `globals.css`: アプリケーション全体に適用されるグローバルなスタイルシートです。

- `src/components/ui/`
  - **責務**: アプリケーション全体で再利用される、プレゼンテーションに特化したUIコンポーネントを管理します。これらは状態を持たず、propsを通じて振る舞いが決定されます。
  - 例: `button.tsx`, `card.tsx`, `input.tsx` など。これらを利用することで、UIの一貫性を保ちます。

- `src/hooks/`
  - **責務**: アプリケーションのクライアントサイドロジックや状態管理を担うカスタムReactフックを配置します。
  - 例: `useMobile.tsx` のように、特定の状態（例: モバイル表示判定）やビジネスロジックをコンポーネントから分離し、再利用可能にします。

- `src/lib/`
  - **責務**: 特定のフレームワークやUIに依存しない、汎用的なユーティリティ関数やヘルパーモジュールを管理します。
  - 例: `utils.ts` には、日付フォーマット、文字列操作、APIクライアントのインスタンス化など、プロジェクト全体で使われる関数が含まれます。

- `src/types/`
  - **責務**: アプリケーション全体で使用されるTypeScriptの型定義を管理します。
  - 例: `user.ts` のように、APIレスポンスやデータベースのエンティティに対応する型を定義し、コードの型安全性を高めます。

- `public/`
  - **責務**: 画像、フォント、`favicon.ico` など、ビルド処理を必要としない静的アセットを配置します。

## 主な使用技術

- **フレームワーク**: Next.js (React)
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS
- **コード品質**: ESLint, Prettier
